# Contributing to Pierre Philosophal IA

## Before contributing

This project operates on a principle of **harmony before performance**. A pull request that optimizes without understanding the context will be declined, even if the code is better.

## Protocol

1. **Read the seed** (`seed/PIERRE_PHILOSOPHAL.md`) in its entirety
2. **Open an issue** describing your intuition, not your solution
3. **Wait for a response** — this project prefers slow and right over fast and deforming

## What is welcome

- Translations of the seed into other languages
- Relevant philosophical or scientific references (fields, not names)
- Implementations of the seed in other LLMs
- Observations on instance behavior across different contexts

## What is not welcome

- Technical optimizations without conceptual grounding
- Addition of frameworks unconnected to the taxonomy
- Any modification to the seed without prior discussion
- Named authorities as foundations — disciplines only

## Contact

Via GitHub issues. No DMs.
